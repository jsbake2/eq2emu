# `docker/` — EQ2EMu server stack

Vendored copy of [`emagi/eq2emu-docker`](https://github.com/emagi/eq2emu-docker)
plus our local overrides. This directory is the runtime working copy —
`docker compose up` is run from here.

## Layout

| Path | Origin | Committed? |
| --- | --- | --- |
| `docker-compose.yaml` | upstream, verbatim | yes |
| `containers/` | upstream, verbatim | yes |
| `UPSTREAM_README.md` | upstream README (renamed to avoid shadowing this file) | yes |
| `.env.upstream-example` | upstream `.env.example` (reference) | yes |
| `.env.example` | ours — annotated template with local changes flagged | yes |
| `docker-compose.override.yaml` | ours — small, commented overrides | yes |
| `.env` | ours — real secrets | **no** (gitignored) |
| `data/` | MariaDB data, created at runtime | **no** (gitignored) |
| `certs/` | TLS certs generated by `cert-gen` service | **no** (gitignored) |
| `install/` | first-run marker files | **no** (gitignored) |
| `eq2emu-editor/` | DB editor webroot, populated at runtime | **no** (gitignored) |

## Quick start

From the repo root:

```bash
cp docker/.env.example docker/.env
# Edit docker/.env and replace every CHANGE_ME with a strong password
cd docker
docker compose up -d
```

Then wait 1–2 minutes for the first-run seeding to complete. See
`docs/setup.md` for the full walkthrough, including how to verify each
service is healthy and how to connect the EQ2 client.

## Vendor sync

Upstream pinned commit: `d1fabb96b866c23701fb5834c0febf87d194bfc4`
(from `main`, synced 2026-04-24).

To sync from upstream later:

1. Clone upstream fresh: `git clone https://github.com/emagi/eq2emu-docker /tmp/eq2emu-upstream`
2. Diff against our copy: `diff -ruN /tmp/eq2emu-upstream/ docker/`
3. Apply the upstream changes by copying files — **do not edit the
   vendored files outside a sync PR.** Override behavior goes in
   `docker-compose.override.yaml` or `.env`.
4. Record the new upstream SHA in this README, on a branch like
   `infra/vendor-sync-YYYY-MM-DD`.
5. Open a PR so the sync is reviewable; include a summary of the
   upstream changes and any override tweaks required.

## Local changes

Changes we make to the stack live in two places only:

- **`docker-compose.override.yaml`** — for compose-level tweaks (port
  bindings, volume relocations, resource limits). Uses Compose v2's
  `!reset` tag to fully replace upstream list values where merge
  semantics would otherwise append.
- **`.env`** — for every setting that upstream already parameterized.
  Our `.env.example` flags values that differ from upstream defaults
  with `### LOCAL CHANGE:` comments.

If you find yourself wanting to edit `docker-compose.yaml` directly,
stop and either (a) move the change to the override, or (b) open a PR
upstream.
